В данном репозитории преставлена игра, что является полной копией игры в репозитории https://github.com/AsyaAp/Lab4_6.git . Функционал и визуал игры полностью перенесены на язык программирования Python с JS, запускается в окне и работает так же, как и на странице браузера. 

Проект состоит из трех основных файлов:

- main.py является главным файлом запуска приложения и содержит класс BartenderGame, который управляет всей игровой логикой. Этот класс контролирует состояние игры, включая отслеживание текущего дня, клиентов и общего прогресса. Он обрабатывает систему диалогов, обеспечивая правильные последовательности реплик и переходы между различными состояниями игры. Важной функцией является проверка рецептов - валидация правильности подобранных ингредиентов и методов приготовления. Класс также ведет статистику успеваемости игрока, отслеживая количество правильных напитков, собранных историй и специальных приготовлений, и на основе этих данных рассчитывает результаты и финальные сцены через систему концовок.
- gui.py реализует графический интерфейс пользователя через класс BartenderGameGUI. Этот класс управляет всем визуальным представлением игры, включая реализацию всех экранов приложения: стартового, обучающего, игрового и финального. Он отвечает за визуализацию барной стойки с ингредиентами и инструментами приготовления, отображение диалоговых окон с персонажами и мыслями бармена. Класс также обеспечивает работу интерактивных компонентов интерфейса, таких как кнопки управления, сворачиваемые панели с рецептами и сетка ингредиентов для составления напитков.
- game_data.py содержит все данные и конфигурацию игры, выступая в роли базы данных, где харится информация персонажей с четырьмя героями, их историями, предпочтениями и диалогами. Также содержится двенадцать рецептов напитков с подробными описаниями ингредиентов и методов приготовления. Дополнительно файл включает системные константы, мысли бармена для различных ситуаций и все текстовые материалы, необходимые для повествования и взаимодействия с игроком. На данный момент я решила сохранить старую версию диалогов и описаний, т.к. для скорости прохождения в процессе тестирования необходимо как можно меньше текста.
